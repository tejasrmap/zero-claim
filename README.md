# 🚀 ZeroClaim AI

### No Claims. No Forms. Instant Protection

---

## 📌 Problem Statement

Gig delivery workers (Zomato, Swiggy) face **unpredictable income loss** due to external factors like:

* 🌧 Heavy rain
* 🌡 Extreme weather
* 🚫 Curfews / disruptions

Currently, there is **no real-time insurance system** to protect their income.
Traditional insurance is slow, manual, and inaccessible.

---

## 💡 Solution

**ZeroClaim AI** is a **parametric insurance platform** that:

* Detects disruptions using real-time data
* Verifies income loss using AI
* Automatically triggers payouts
* Eliminates claim filing completely

👉 **No paperwork. No delays. Instant payouts.**

---

## 👤 Target Users

* Food delivery partners (Zomato, Swiggy)
* Gig workers dependent on daily earnings

---

## ⚙️ How It Works

1. User subscribes to a weekly insurance plan
2. System monitors:

   * Weather data
   * Delivery activity
3. AI detects income drop
4. Parametric trigger activates
5. Instant payout credited

## 🤖 AI/ML Approach

Our AI engine uses **multi-factor validation**:

* 🌧 Weather intensity (rainfall levels)
* 📉 Order drop percentage
* 📍 Location correlation
* 📊 Historical earning patterns

👉 Decision Rule Example:

> If Rain > 70mm AND Orders drop > 50% → Trigger claim

# 🚨 Adversarial Defense & Anti-Spoofing Strategy

---

## 1️⃣ Differentiation (Real vs Fake Users)

We differentiate genuine vs spoofed users using:

* 📡 Device motion patterns (is the rider actually moving?)
* 📶 Network signal fluctuations (real outdoor vs indoor spoofing)
* 📍 Multi-source location validation (GPS + network + IP)
* 🧠 Behavioral consistency (historical riding patterns)

👉 A spoofed user will show **static patterns**, unlike real riders.

---

## 2️⃣ Data Points Used

Beyond GPS, we analyze:

* Accelerometer data (movement)
* App activity logs
* Delivery acceptance patterns
* Time-based earning consistency
* Weather + city-wide activity trends

👉 Fraud rings show **cluster anomalies**

---

## 3️⃣ UX Balance (Fairness System)

We ensure **honest users are not penalized**:

* ⚠ Flagged claims → “Under Review” (not rejected)
* ⏳ Temporary partial payout
* 👤 Manual verification fallback
* 📊 Confidence score before blocking

👉 This ensures **trust + fairness**
